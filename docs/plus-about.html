plus使用经验

  1. plus对象是对window对象的扩展。
  2. 使用plus对象最需要注意的是不要在页面初始化期间使用。就算mui.plusReady方法执行完成之后也不行。
  3. 因为mui.plusReady是异步方法，在其之后的代码有可能在其之前执行。
  4. 每个页面附加的js代码应该遵循一个基本的结构。
    1. 调用mui.init(...)，初始化mui对象，配置本页面的一些特殊之处，例如是否响应一些消息,调整mui的默认值，预加载子页面等等。
    2. 调用mui.plusReady(...)，初始化plus对象，修改window对象，例如注册window对象专有的消息响应函数等等。
    3. window对象专有的消息的列表。
    4. 定义信号变量。
    5. 定义各页面元素的消息响应函数。各个函数可以通过信号变量协同工作。
    6. 绑定各页面元素的消息响应函数。
  5. 在各页面元素的消息响应函数的定义中，可以使用plus对象，因为他们并不是在页面初始化或加载期间执行。
  6. 当用户操作界面，触发消息，调用响应函数的时候，plus对象应该已经准备完毕，可以使用了。
